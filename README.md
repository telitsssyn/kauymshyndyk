# Сайт Кауым-Церкви Шындык

Сайт церкви в Павлодаре: расписание богослужений, новости и события, страница для новичков, служения, пожертвования и контакты. Управление контентом — через встроенную админку (Payload CMS) на `/admin`.

Инструкция для контент-менеджера: [docs/ADMIN.md](docs/ADMIN.md).

## Стек

- **Next.js (App Router) + TypeScript** — фронтенд и рендеринг
- **Payload CMS 3** — админка и контент, встроена в тот же проект
- **PostgreSQL** — база данных (локально `embedded-postgres`, на проде — Neon)
- **Tailwind CSS 4** — стили
- **next-intl** — переводы интерфейса; сейчас только `ru`, казахский/английский добавляются словарём и строкой в конфиге
- Хостинг: **Vercel** (бесплатный тариф), изображения — **Vercel Blob**

## Запуск локально

Нужен Node.js 20+.

```bash
npm install

# 1. Поднять локальную базу данных (терминал 1, оставить работающим)
npm run dev:db

# 2. Заполнить базу стартовыми заглушками и создать администратора (один раз)
npm run seed

# 3. Запустить сайт (терминал 2)
npm run dev
```

- Сайт: http://localhost:3000
- Админка: http://localhost:3000/admin (после `npm run seed`: `admin@example.com` / `admin12345` — смените пароль)

Полезные команды:

| Команда | Что делает |
| --- | --- |
| `npm run dev:db` | локальный PostgreSQL на порту 5433 (данные в `.pgdata/`) |
| `npm run seed` | заполняет пустую базу заглушками, создаёт админа |
| `npm run generate:types` | обновляет `src/payload-types.ts` после изменения коллекций |
| `npm run lint` / `npx tsc --noEmit` | линт и проверка типов |
| `npm run build` | production-сборка |

## Структура

```
src/
  app/(frontend)/[locale]/   страницы сайта (маршруты с префиксом локали /ru/...)
  app/(payload)/             админка и API Payload
  collections/               News, Ministers, GalleryItems, Media, Users
  globals/                   Настройки, Расписание и страницы с фиксированной структурой
  components/                компоненты интерфейса
  i18n/                      локали, словари (messages/ru.json), маршруты
  lib/                       запросы к CMS, расчёт ближайших служб, слаги
  seed/                      скрипт стартового наполнения
```

Публичные адреса страниц задаются в `src/i18n/routing.ts` (например, внутренний `/schedule` → `/ru/raspisanie`).

## Деплой (Vercel + Neon + Blob, всё бесплатно)

1. **База данных.** Создайте проект на [neon.tech](https://neon.tech) (регион ближе к Казахстану, например Frankfurt). Скопируйте **Pooled connection string**.
2. **Репозиторий.** Запушьте код на GitHub.
3. **Vercel.** [vercel.com](https://vercel.com) → Add New Project → импортируйте репозиторий. В настройках проекта:
   - Build Command: `npm run ci`
   - Environment Variables:
     | Имя | Значение |
     | --- | --- |
     | `DATABASE_URL` | pooled-строка Neon |
     | `PAYLOAD_SECRET` | длинная случайная строка (например, `openssl rand -hex 32`) |
     | `NEXT_PUBLIC_SERVER_URL` | `https://ваш-домен.vercel.app` (после первого деплоя обновите на постоянный домен) |
4. **Хранилище изображений.** В проекте Vercel: Storage → Create → Blob. Переменная `BLOB_READ_WRITE_TOKEN` добавится автоматически — переделайте деплой (Redeploy), чтобы она подхватилась.
5. **Миграции.** Перед первым деплоем создайте миграцию локально и закоммитьте её:
   ```bash
   npm run payload migrate:create initial
   ```
   Команда `npm run ci` на Vercel применяет миграции (`payload migrate`) и собирает сайт. При каждом изменении коллекций/глобалов создавайте новую миграцию той же командой.
6. **Первый вход.** Откройте `https://ваш-домен/admin` — Payload предложит создать первого администратора. Заполните «Настройки сайта», «Расписание» и остальные разделы (см. [docs/ADMIN.md](docs/ADMIN.md)).

> Локальная разработка использует режим `push` (таблицы создаются автоматически), поэтому `payload migrate` локально запускать не нужно — только `migrate:create` для прода.

## Как добавить казахский или английский язык

1. `src/i18n/routing.ts` — добавить локаль в `locales` и адреса страниц в `pathnames`.
2. `src/i18n/messages/kk.json` — создать словарь по образцу `ru.json`.
3. `src/payload.config.ts` — добавить локаль в `localization.locales`.
4. В админке появится переключатель языка у локализованных полей.
